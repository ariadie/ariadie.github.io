Non-dominated Sorting Genetic Algorithm II

NSGA-II diperkenalkan oleh Deb, Pratap, Agarwal, & Meyarivan (2002) sebagai pengembangan dari NSGA (Srinivas & Deb, 1994). Algoritma ini mengintegrasikan non-dominated sorting, crowding distance, dan strategi elitism berbasis (μ+λ) untuk menyelesaikan masalah optimasi multiobjective secara efisien.

📊

Non-dominated Sorting

Populasi diurutkan ke dalam front-front Pareto berdasarkan dominasi. Front pertama (F₁) berisi solusi nondominated terbaik, lalu F₂, dan seterusnya.

📏

Crowding Distance

Mengestimasi kepadatan solusi di sekitar setiap individu dalam satu front. Semakin besar jarak, semakin baik untuk menjaga keragaman populasi.

🏆

Elitism (μ+λ)

Populasi induk dan offspring digabungkan, lalu diseleksi berdasarkan rank (front) dan crowding distance — menjamin solusi terbaik tidak hilang.

Langkah-langkah Algoritma NSGA-II

NSGA-II menggabungkan orangtua dan offspring, lalu menyeleksi N individu terbaik dari gabungan tersebut.

  1. Inisialisasi Bangkitkan populasi awal \(P_0\) berukuran \(N\). Tetapkan \(t = 0\).
  2. Non-dominated Sorting Urutkan \(P_t\) ke dalam front Pareto \(F_1, F_2, \dots, F_k\) berdasarkan relasi dominasi. Individu di front lebih rendah memiliki rank lebih baik.
  3. Crowding Distance Hitung crowding distance untuk setiap individu dalam setiap front. Nilai ini mengukur kepadatan di sekitar solusi dalam ruang objektif.
  4. Binary Tournament Selection Seleksi orangtua menggunakan crowded tournament: pilih individu dengan rank lebih rendah; jika sama, pilih crowding distance lebih besar.
  5. Variasi (Crossover & Mutation) Terapkan operator simulated binary crossover (SBX) dan polynomial mutation pada pasangan orangtua untuk menghasilkan offspring \(Q_t\) berukuran \(N\).
  6. Seleksi Lingkungan (μ+λ) Gabungkan \(R_t = P_t \cup Q_t\) (ukuran \(2N\)). Urutkan \(R_t\) dengan non-dominated sorting, lalu isi generasi baru \(P_{t+1}\) dengan front terbaik berdasarkan rank dan crowding distance hingga mencapai \(N\) individu. Jika \(t < T_{\max}\), \(t = t+1\) dan ulangi dari langkah 2.

Pseudo-code Ringkas

Input:  N (pop size), T (max gen)
Output: P (final population)

1.  \(P_0\) ← init_population(N)
2.  for t = 0 to T-1:
3.    \(Q_t\) ← make_offspring(\(P_t\))  // SBX + polynomial mutation
4.    \(R_t\) ← \(P_t \cup Q_t\)
5.    F ← non_dominated_sort(\(R_t\))
6.    \(P_{t+1}\) ← ∅
7.    for each front \(F_i\) in F:
8.      if |\(P_{t+1}\)| + |\(F_i\)| ≤ N:
9.        \(P_{t+1}\) ← \(P_{t+1} \cup F_i\)
10.     else:
11.       crowding_distance(\(F_i\))
12.       \(P_{t+1}\) ← \(P_{t+1} \cup\) select_best(\(F_i\), N - |\(P_{t+1}\)|)
13.       break
14. return \(P_T\)
