Ikhtisar

Bab ini membahas pencarian di lingkungan yang sebagian terobservasi (partially observable) dan pencarian daring (online search). Agen tidak selalu memiliki informasi lengkap tentang keadaan lingkungan—ia harus membuat keputusan di tengah ketidakpastian.

1 Sensorless (Conformant) Planning

Agen tidak memiliki sensor sama sekali—harus menemukan rencana yang berhasil untuk semua kemungkinan keadaan awal.

2 Belief State

Alih-alih melacak keadaan nyata, lacak himpunan keadaan yang mungkin (belief state) yang merepresentasikan ketidakpastian agen.

3 Online Search

Agen harus bertindak tanpa mengetahui lingkungan sepenuhnya—mengeksplorasi sekaligus mencari tujuan secara langsung (real-time).

Sensorless (Conformant) Planning

Dalam sensorless planning, agen tidak memiliki sensor sama sekali. Ia harus menemukan satu urutan aksi (conformant plan) yang berhasil untuk semua kemungkinan keadaan awal.

\[ \text{Conformant plan } \pi \text{ succeeds if } \forall s_0 \in \text{InitialBelief}, \text{Result}(s_0, \pi) \subseteq \text{GoalSet} \]

Contoh: Robot Tanpa Sensor

Bayangkan robot yang harus mencapai ruangan tertentu, tetapi tidak mengetahui posisi awalnya. Robot harus merencanakan gerakan yang pasti akan membawanya ke tujuan, apa pun posisi awalnya.

• Pendekatan BFS pada Belief State

Gunakan BFS pada ruang keadaan keyakinan (belief-state space). Setiap node adalah himpunan keadaan, bukan keadaan tunggal.

• Kompleksitas

Waktu dan ruang eksponensial terhadap jumlah variabel keadaan, karena jumlah belief state bisa mencapai \(2^n\) untuk \(n\) boolean variabel.

Penting: Sensorless planning lebih sulit dari planning sensor lengkap karena ruang keadaan keyakinan secara eksponensial lebih besar dari ruang keadaan asli.

Belief States (Keadaan Keyakinan)

Belief state adalah himpunan keadaan nyata yang mungkin berdasarkan pengamatan agen sejauh ini. Agen beralih antar belief state melalui fungsi transisi keyakinan:

\[ \hat{T}(\hat{b}, a) = \bigcup_{s \in \hat{b}} \text{Trans}(s, a) \] Transisi keyakinan
\[ \hat{T}(\hat{b}, e, a) = \{s' : s' \in \text{Trans}(s, a) \text{ dan } \text{Obs}(s', e) \} \] Update dengan pengamatan

Properti Belief State

PropertiPenjelasan
Initial belief state\(\hat{b}_0 = \{s_0\}\) jika diketahui, atau himpunan semua keadaan mungkin jika tidak
Goal test\(\hat{b} \subseteq \text{GoalSet}\) — semua keadaan dalam keyakinan adalah tujuan
Transition functionGabungkan hasil transisi dari setiap keadaan dalam himpunan keyakinan
ObservationPersempit keyakinan: buang keadaan yang tidak konsisten dengan pengamatan
Peringatan: Belief state space bisa sangat besar. Untuk \(n\) variabel boolean, ada \(2^n\) kemungkinan belief state. Pruning dan heuristik sangat diperlukan.

Contoh: Sensor Toggle

Pertimbangkan robot di lorong 5 sel. Sensor aktif/Nonaktif secara acak. Tanpa sensor, agen harus merencanakan berdasarkan belief state—himpunan kemungkinan posisi robot.

Online Search (Pencarian Daring)

Dalam online search, agen tidak mengetahui model transisi lingkungan (atau hanya sebagian). Ia harus mengeksplorasi lingkungan secara langsung, bertindak, dan memperbarui pengetahuannya secara simultan.

→ Offline Search

Agen mengetahui model lingkungan sepenuhnya. Dapat merencanakan jalur lengkap sebelum bertindak. Contoh: A*, Dijkstra.

→ Online Search

Agen mengeksplorasi selama bertindak. Tidak bisa merencanakan jalur lengkap sebelumnya. Contoh: LRTA*, RTAA*.

LRTA* (Learning Real-Time A*)

LRTA* adalah algoritma online search yang mempelajari biaya keadaan dari pengalaman langsung. Setiap kali agen mengunjungi suatu keadaan, ia memperbarui estimasi biaya berdasarkan pengamatan tetangga yang sebenarnya.

\[ \hat{h}(s) \leftarrow \max_{a \in \text{Actions}(s)} \big[ c(s,a) + \hat{h}(\text{Result}(s,a)) \big] \] Update LRTA*

Algoritma LRTA*

function LRTA-Search(problem) returns aksi

current ← problem.InitialState

for each s in problem.States do

\(\hat{h}(s) \leftarrow\) heuristic(s) or 0

repeat forever

s ← current

for each a in Actions(s) do

\(\hat{h}(s) \leftarrow \min(\hat{h}(s), c(s,a) + \hat{h}(\text{Result}(s,a)))\)

a* ← argmin\(_a [c(s,a) + \hat{h}(\text{Result}(s,a))]\)

current ← Result(s, a*)

return a*

Properti LRTA*

PropertiNilai
KelengkapanYa (pada graf berhingga)
OptimalitasTidak — bisa menjalankan lebih dari \(n^2\) langkah
BiayaBatas atas: \((n - 1) \times n^2 \times c_{\max}\)
Ruang memori\(O(n)\) untuk menyimpan \(\hat{h}\) setiap keadaan
Perhatian: LRTA* bisa menjadi sangat tidak efisien untuk graf dengan banyak keadaan. Gunakan RTAA* dengan bukti batas atas yang lebih ketat untuk graf berhingga.

Latihan Terselesaikan

Latihan 1 — Sensorless Planning

Robot di lorong 3 sel (A, B, C). Robot tidak mengetahui posisi awalnya. Dari sel manapun, robot bisa bergerak ke kiri atau ke kanan (jika tidak ada dinding). Tuliskan conformant plan untuk memastikan robot mencapai sel B.

Tampilkan solusi

Solusi: Rencana: [Kanan, Kiri].

Analisis:

  • Jika mulai di A: Kanan → B. B adalah tujuan.
  • Jika mulai di B: Kanan → C, lalu Kiri → B.
  • Jika mulai di C: Kanan → B (diasumsikan looping/bounce).

Rencana ini berhasil untuk semua posisi awal karena B adalah sel tengah—bergerak ke kanan sekali dari A mencapai B, dan bergerak ke kanan dari C kemudian kembali ke B.

Latihan 2 — Belief State Transisi

Sebuah agen memiliki 3 keadaan: \(\{S_1, S_2, S_3\}\). Belief state awal: \(\{S_1, S_2\}\). Setelah aksi \(a\), transisi: \(S_1 \to S_2\), \(S_2 \to S_3\), \(S_3 \to S_1\). Apakah belief state setelah aksi \(a\)?

Tampilkan solusi

Belief state setelah aksi: \(\hat{T}(\{S_1, S_2\}, a) = \{\text{Result}(S_1, a), \text{Result}(S_2, a)\} = \{S_2, S_3\}\).

Prinsip: gabungkan hasil transisi dari setiap keadaan dalam belief state. \(S_1\) menjadi \(S_2\), \(S_2\) menjadi \(S_3\).

Latihan 3 — LRTA* Update

Pada graf sederhana: A terhubung ke B (biaya 1) dan C (biaya 4). B terhubung ke D (biaya 2). C terhubung ke D (biaya 1). D adalah tujuan. \(\hat{h}(A) = 3\), \(\hat{h}(B) = 2\), \(\hat{h}(C) = 2\), \(\hat{h}(D) = 0\). Apa yang dilakukan LRTA* saat berada di A?

Tampilkan solusi

Langkah LRTA* di A:

  1. Untuk aksi \(a_1\) (ke B): \(c(A,B) + \hat{h}(B) = 1 + 2 = 3\)
  2. Untuk aksi \(a_2\) (ke C): \(c(A,C) + \hat{h}(C) = 4 + 2 = 6\)
  3. Update: \(\hat{h}(A) \leftarrow \min(3, 6) = 3\) (tidak berubah)
  4. Pilih aksi \(a_1\) (ke B), karena lebih murah
  5. Update \(\hat{h}(A) = \min(3, c(A,B) + \hat{h}(B)) = \min(3, 3) = 3\)

Agen bergerak ke B. Di B, ia akan menghitung: \(\hat{h}(B) = c(B,D) + \hat{h}(D) = 2 + 0 = 2\). Tidak ada update.

Latihan 4 — Online vs Offline

Kapan Anda akan menggunakan LRTA* dibandingkan A*? Berikan contoh skenario untuk masing-masing.

Tampilkan solusi

A* (Offline): Gunakan ketika peta lingkungan sudah diketahui sepenuhnya. Contoh: menghitung rute terpendek dari rumah ke kantor menggunakan GPS.

LRTA* (Online): Gunakan ketika lingkungan baru dan tidak diketahui. Contoh: robot penjelajah Mars yang harus bergerak tanpa peta, menemukan rintangan secara langsung, dan belajar dari setiap langkah.

Ringkasan

Poin-Poin Penting

✓
Sensorless Planning: Agen tanpa sensor harus menemukan rencana yang berhasil untuk semua keadaan awal yang mungkin.
✓
Belief State: Himpunan keadaan yang mungkin — merepresentasikan ketidakpastian agen. Transisi keyakinan: gabungkan hasil dari setiap keadaan dalam himpunan.
✓
Online Search: Agen bertindak tanpa mengetahui model lingkungan sepenuhnya. Eksplorasi dan perencanaan terjadi secara bersamaan.
✓
LRTA*: Algoritma online yang mempelajari estimasi biaya dari pengalaman. Lengkap pada graf berhingga, tapi tidak optimal.
✓
Trade-off: Offline search optimal tapi butuh pengetahuan lengkap. Online search aman dan adaptif tapi bisa tidak efisien.
