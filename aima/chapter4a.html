Ikhtisar Nondeterministic AND-OR Search Contingency Plans Simulasi Latihan Ringkasan

Ikhtisar

Pencarian klasik (Bab 3) mengasumsikan lingkungan yang deterministik, sensorik lengkap, dan tunggal-keadaan. Bab ini menghapus asumsi-asumsi tersebut. Ketika tindakan memiliki beberapa hasil yang mungkin, agen memerlukan strategi yang lebih kaya daripada urutan tindakan sederhana.

1 Nondeterministic Actions

Tindakan memiliki hasil yang tidak pasti — satu aksi dapat mengarah ke beberapa keadaan. Contoh: ban bocor di vacuum world, robot navigasi di terrain berpasir.

2 AND-OR Search

Algoritma untuk mencari solusi di lingkungan nondeterministik. Menghasilkan pohon keputusan dengan node OR (pilihan agen) dan node AND (hasil lingkungan).

3 Contingency Plans

Berbeda dari urutan aksi, rencana kontingensi adalah pohon yang menangani berbagai hasil — dengan kata lain, rencana if-then-else.

Mengapa ini penting? Di dunia nyata, robot menghadapi sensor yang bising, aksi yang gagal, dan lingkungan yang berubah. Rencana klasik yang rapuh akan hancur; rencana kontingensi yang adaptif akan bertahan.

Nondeterministic Actions

Pada lingkungan nondeterministik, fungsi Result(s, a) mengembalikan himpunan keadaan yang mungkin, bukan satu keadaan tunggal.

\[ \text{Result}(s, a) = \{s_1, s_2, \ldots, s_k\} \]

Contoh: Vacuum World (Ban Bocor)

Bayangkan vacuum world di mana robot memiliki ban bocor. Ketika robot mencoba bergerak, ada kemungkinan:

  • 80% — robot bergerak dengan sukses
  • 20% — robot tetap di tempat (aksi gagal)
AksiKeadaan AwalHasil Mungkin
Right[A, Clean]{[B, Clean], [A, Clean]}
Left[B, Dirty]{[A, Dirty], [B, Dirty]}
Suck[A, Dirty]{[A, Clean]} (deterministik)
Suck[A, Clean]{[A, Clean]} (sudah bersih)

Perbedaan dengan Deterministik

D Deterministik

\(\text{Result}(s,a) = \{s'\}\) — satu hasil. Solusi = urutan aksi: \([a_1, a_2, \ldots, a_n]\).

N Nondeterministik

\(\text{Result}(s,a) = \{s_1, s_2, \ldots\}\) — banyak hasil. Solusi = pohon keputusan (contingency plan).

Perhatikan: Solusi sederhana seperti \([Right, Suck, Left]\) tidak cukup untuk lingkungan nondeterministik! Jika Right gagal, agen tidak tahu harus berbuat apa. Diperlukan rencana yang lebih kaya.

AND-OR Search

AND-OR search menghasilkan pohon keputusan (bukan jalur sederhana). Pohon ini memiliki dua jenis node:

OR OR Nodes

Pilihan agen. Agen memilih satu aksi dari beberapa alternatif. Setiap anak mewakili satu pilihan.

AND AND Nodes

Hasil lingkungan. Lingkungan memilih hasil. Semua anak harus ditangani oleh rencana.

Algoritma AND-OR-GRAPH-SEARCH

Fungsi OR-GRAPH-SEARCH(problem) mengembalikan pohon keputusan, atau failure.

\[ \text{OR-GRAPH-SEARCH}(problem) \Rightarrow \text{pohon keputusan atau failure} \]
LangkahDeskripsi
1Inisialisasi tree dengan node awal problem.INITIAL-STATE
2Inisialisasi frontier dengan node-root
3Loop: pilih node leaf dari frontier
4Jika leaf = goal: return tree
5Jika leaf = failure: backtrack, hapus subtree
6Expand leaf: tambahkan AND node untuk setiap aksi, OR nodes untuk setiap hasil
7Periksa redundant path — jika state sudah ada di tree, jangan duplikasi

Struktur Pohon AND-OR

Aturan pembacaan:
  • Node OR (lingkaran): agen memilih salah satu anak
  • Node AND (persegi): lingkungan memilih, semua anak harus ditangani
  • Daun goal: keadaan tujuan tercapai
  • Daun failure: jalur ini tidak valid

Contoh: Vacuum World dengan Ban Bocor

Pada keadaan awal \([A, Dirty]\), agen ingin membersihkan kedua ruangan:

                    
            OR: [A, Dirty]
                |
            ┌─────────────┐
            |             |
        OR: Suck     OR: Right
            |             |
        ┌───────┐     ┌────────────┐
        |       |     |            |
    AND:Suck  AND:Right    OR: Suck  OR: Right
        |       |     |            |
    [A,Clean]  AND   [A,Clean]   [A,Clean] / [B,Dirty]
                    |    |
                [A,Clean] [B,Clean] (goal!)

Contingency Plans (Rencana Kontingensi)

Rencana kontingensi adalah pohon keputusan yang memberikan instruksi untuk setiap kemungkinan hasil. Berbeda dari rencana sekuensial sederhana.

> Rencana Sekuensial

Jalur linier: \([a_1, a_2, \ldots, a_n]\). Hanya berfungsi untuk lingkungan deterministik.

>< Rencana Kontingensi

Pohon keputusan: setiap node AND memiliki cabang untuk setiap hasil. Berfungsi untuk nondeterministik.

Sifat Penting

Contingent PlanPohon keputusan lengkap yang menangani semua kemungkinan
Sensorless (Conformant)Rencana untuk lingkungan tanpa sensor — harus bekerja untuk semua keadaan awal

Latihan

Latihan 1 — Identifikasi Tipe Pohon

Pada pohon keputusan berikut, identifikasi node OR dan AND. Apakah rencana ini lengkap untuk menangani semua kemungkinan?

        OR: Start
        ├── Right (OR)
        │   ├── AND (result)
        │   │   ├── [B, Clean] (goal) ✓
        │   │   └── [A, Dirty] (failure) ✗
        └── Suck (OR)
            └── AND (result)
                └── [A, Clean] → Right → ...
Tampilkan solusi

Analisis:

  • Root "Start" adalah OR node — agen memilih Right atau Suck
  • "Right" dan "Suck" adalah OR nodes — mewakili pilihan aksi
  • "AND (result)" adalah AND node — lingkungan menghasilkan beberapa kemungkinan
  • Rencana TIDAK lengkap karena cabang Right menghasilkan [A, Dirty] (failure) yang tidak ditangani

Untuk lengkap, agen harus menambahkan Suck setelah Right gagal, atau memilih Suck terlebih dahulu.

Latihan 2 — Redundant Path

Dalam AND-OR search, mengapa penting memeriksa redundant path?

Tampilkan solusi

Redundant path terjadi ketika state yang sama muncul di cabang tree yang berbeda tanpa penanganan ulang. Tanpa deteksi redundansi, tree bisa tumbuh eksponensial dengan duplikasi state. AND-OR graph search menyimpan setiap state hanya satu kali dan mereferensikan subtree yang sudah ada, mengubah tree menjadi DAG (Directed Acyclic Graph).

Contoh: Jika Right dari A menghasilkan [B, Clean], dan Suck dari [B, Clean] → [B, Clean] (sudah bersih), kita tidak perlu membuat subtree baru untuk [B, Clean] yang kedua kalinya.

Latihan 3 — Optimal Contingency Plan

Bagaimana cara menentukan rencana kontingensi yang optimal?

Tampilkan solusi

Rencana kontingensi optimal adalah yang meminimalkan expected cost (biaya harapan) atau worst-case cost tergantung kriteria:

  • Optimal expected cost: Rata-rata berbobot berdasarkan probabilitas setiap hasil
  • Optimal worst-case: Meminimalkan biaya jalur terpanjang (minimum-maksimum)

Contoh: Jika Right (prob 0.8) mencapai goal dengan cost 1, dan Suck (prob 0.2) mencapai goal dengan cost 3, expected cost = 0.8×1 + 0.2×3 = 1.4.
