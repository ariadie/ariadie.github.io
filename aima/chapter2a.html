Ikhtisar Tipe Agen Agent Function Rasionalitas Lingkungan Latihan

Ikhtisar

Bab ini memperdalam konsep agen cerdas yang diperkenalkan di Bab 1. Kita akan memahami apa itu agen, bagaimana agen berinteraksi dengan lingkungan, dan apa yang membuat agen bersifat rasional.

A Agen (Agent)

Entitas yang mempersepsi lingkungan melalui sensors dan bertindak melalui actuators. Manusia sendiri adalah agen: mata/ telinga = sensors, tangan/kaki = actuators.

PE Lingkungan

Segala sesuatu di luar agen. Didefinisikan oleh sifat-sifatnya: observable vs partial, deterministic vs stochastic, episodic vs sequential, dan lain-lain.

f Agent Function

Fungsi abstrak yang memetakan percept sequence ke aksi: \(f: P^* \to A\). Implementasinya adalah agent program yang berjalan pada arsitektur fisik.

2.1 Tipe-Tipe Agen

Agen dapat dikelompokkan berdasarkan kompleksitasnya:

1 Human Agent

Sensor: mata, telinga, hidung, kulit (sentuh). Actuators: mulut, tangan, kaki, organ tubuh lainnya.

2 Robotic Agent

Sensor: kamera, LiDAR, sonar, inframerah, accelerometer. Actuators: motor, roda, lengan robotik, gripper.

3 Software Agent

Sensor: keystrokes, file contents, network data, database queries. Actuators: display, file writes, network packets, API calls.

Penting: Perbedaan antara "agen" dan "program" bersifat gradasi, bukan kategoris tajam. Semua agen cerdas adalah program, tetapi tidak semua program adalah agen — program harus memiliki akses ke sensors dan actuators untuk berinteraksi dengan lingkungan.

2.2 Agent Function

Agent function adalah entitas abstrak yang merupakan spesifikasi teknis dari agen. Ia memetakan setiap percept sequence ke aksi:

\[ f: P^* \to A \]

\(P^*\) = himpunan semua kemungkinan percept sequence (tak hingga)
\(A\) = himpunan aksi yang tersedia

Agent Program ≠ Agent Function

Agent function adalah spesifikasi matematika (tabel tak hingga). Agent program adalah implementasi konkret yang berjalan pada arsitektur agen — sebuah program komputer yang merealisasikan agent function.

Contoh: Agent Tabel Sederhana

Misalnya agen vacuum sederhana dengan 2 lokasi (A, B) dan 2 kondisi (bersih/kotor):

Percept SequenceAksi
\([Loc_A, Clean]\)Right
\([Loc_A, Dirty]\)Suck
\([Loc_B, Clean]\)Left
\([Loc_B, Dirty]\)Suck
\([Loc_A, Clean, Loc_B, Clean]\)Right
... (tak hingga kombinasi)...
Masalah: Jumlah kemungkinan percept sequence adalah \(|P|^n\) di mana \(n\) = panjang sequence. Untuk vacuum dengan 2 lokasi dan 2 kondisi: \(4^1 + 4^2 + 4^3 + \ldots\) — tak hingga! Kita butuh pendekatan yang lebih cerdas daripada tabel tak hingga.

2.3 Agen Rasional

Agen rasional adalah agen yang melakukan tindakan yang benar. "Tindakan yang benar" didefinisikan berdasarkan performance measure yang telah ditentukan — bukan berdasarkan apa yang agen "pikirkan" atau "inginkan".

\[ \text{agent}^* = \arg\max_{a \in A} \sum_{t=1}^{\infty} U_t \cdot \pi_t(a) \]

Di mana \(U_t\) = utilitas dari timestep \(t\), \(\pi_t(a)\) = probabilitas aksi \(a\) pada waktu \(t\).

Kondisi Rasionalitas

P Performance Measure

Kriteria yang menentukan keberhasilan. Harus ditentukan oleh pembuat desain, bukan oleh agen. Berdasarkan apa yang terjadi di lingkungan, bukan berdasarkan apa yang dipikirkan agen.

E Persepsi Lingkungan

Agen harus memiliki akses ke seluruh riwayat persepsi untuk membuat keputusan yang benar. Momen saat ini saja tidak cukup untuk banyak masalah.

K Pengetahuan Agen

Agen harus memiliki pengetahuan tentang bagaimana lingkungan bekerja — termasuk model transisi, aturan, dan hukum fisik yang relevan.

A Aksi Tersedia

Daftar aksi yang dapat dilakukan agen di setiap keadaan. Aksi ini harus komplit dan representatif untuk semua kemungkinan tindakan.

Rasionalitas vs Omniscience

Kunci: Rasionalitas bukan omniscience. Agen rasional tidak harus tahu segalanya — ia harus membuat keputusan terbaik dengan informasi yang tersedia. Jika informasi tambahan dapat diperoleh dengan biaya lebih rendah dari nilainya, agen rasional akan memperolehnya.

Autonomous vs Non-Autonomous

Agen yang keputusannya bergantung hanya pada persepsi sendiri (tanpa campur tangan eksternal) disebut autonomous. Sebaliknya, agen yang didesain oleh penciptanya (seperti GPS navigator) bersifat non-autonomous.

2.4 Sifat-Sifat Lingkungan

Pemilihan agen yang tepat memerlukan pemahaman tentang sifat lingkungan. Berikut adalah properti-properti kunci:

1. Fully Observable vs Partially Observable

Fully: Sensor agen mendeteksi semua aspek dari keadaan lingkungan. Partial: Agen hanya melihat sebagian keadaan (perlu model internal).

2. Deterministic vs Stochastic

Deterministic: Hasil aksi selanjutnya sepenuhnya ditentukan oleh keadaan saat ini dan aksi yang dilakukan. Stochastic: Hasil aksi melibatkan ketidakpastian (probabilistik).

3. Episodic vs Sequential

Episodic: Pengalaman agen terbagi dalam episode terpisah — aksi di satu episode tidak mempengaruhi episode selanjutnya. Sequential: Keputusan saat ini mempengaruhi masa depan.

4. Static vs Dynamic vs Semidynamic

Static: Lingkungan tidak berubah saat agen berpikir. Dynamic: Lingkungan berubah secara real-time. Semidynamic: Lingkungan berubah, tapi performance measure tidak.

5. Discrete vs Continuous

Discrete: Keadaan dan aksi berjumlah terbatas/hitung. Continuous: Keadaan dan aksi bernilai kontinu (real number).

6. Single Agent vs Multiagent

Single: Agen beroperasi sendiri. Multiagent: Ada beberapa agen yang berinteraksi — bisa kooperatif atau kompetitif.

Latihan Terselesaikan

Latihan 1 — Agent Function vs Agent Program

Jelaskan perbedaan antara agent function dan agent program. Berikan analogi.

Tampilkan solusi

Agent function adalah spesifikasi matematika abstrak — tabel tak hingga yang memetakan setiap kemungkinan percept sequence ke aksi. Agent program adalah implementasi konkret yang berjalan pada arsitektur fisik — sebuah program komputer yang merealisasikan agent function (mungkin secara parsial, untuk sequence yang mungkin dihadapi).

Analogi: Agent function seperti "buku resep lengkap" (tak hingga), sedangkan agent program seperti "koki ahli" yang menghasilkan masakan serupa tanpa harus membaca seluruh buku resep.

Latihan 2 — Sifat Lingkungan

Sebutkan minimal 3 contoh lingkungan untuk masing-masing kategori: (a) fully observable, (b) partially observable, (c) deterministic, (d) stochastic.

Tampilkan solusi

Fully observable: Chess, Tic-tac-toe, Image classification, Sudoku.

Partially observable: Poker, Autonomous driving, Medical diagnosis, Trading stock.

Deterministic: Chess, Tic-tac-toe, Sudoku, Math problems.

Stochastic: Poker, Backgammon, Weather forecasting, Robotics (real world).

Latihan 3 — Mengapa Performance Measure Ditentukan Pembuat Desain?

Mengapa performance measure harus ditentukan oleh pembuat desain, bukan oleh agen sendiri?

Tampilkan solusi

Jika agen yang menentukan performance measure-nya sendiri, agen dapat memanipulasi definisi "keberhasilan" untuk mempermudah pencapaiannya. Performance measure harus objektif dan tercermin dari apa yang benar-benar penting bagi pengguna/pembuat desain. Ini memastikan agen bekerja untuk kepentingan manusia, bukan kepentingannya sendiri.
