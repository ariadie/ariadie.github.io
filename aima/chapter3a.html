Ikhtisar Formulasi Masalah BFS DFS UCS IDS Simulasi Latihan

Ikhtisar

Bab ini membahas bagaimana agen cerdas memecahkan masalah melalui pencarian (search). Agen berbasis tujuan (goal-based agents) menentukan serangkaian aksi yang mengubah keadaan lingkungan dari kondisi awal menuju kondisi yang diinginkan.

1 Problem Formulation

Mendefinisikan masalah secara formal: keadaan awal, ruang keadaan, aksi, model transisi, pengujian tujuan, dan biaya jalur.

2 Uninformed Search

Pencarian tanpa informasi heuristik: BFS, DFS, Uniform-Cost, dan Iterative Deepening—hanya menggunakan definisi masalah.

3 Completeness & Optimality

Algoritma yang lengkap menjamin solusi ditemukan; yang optimal menjamin jalur terbaik (biaya minimum).

Formulasi Masalah (Problem Formulation)

Sebuah masalah P didefinisikan sebagai 6-tuple: P = ⟨S, A, T(s,a), GoalTest(s), StepCost(s,a,s′)⟩

S State Space (Ruang Keadaan)

Himpunan semua keadaan yang mungkin. Contoh: kota-kota di peta Rumania. Disediakan oleh agent program — tidak harus keadaan dunia sesungguhnya.

s0 Initial State (Keadaan Awal)

Keadaan tempat agen memulai. Contoh: s0 = Arad.

A Actions (Aksi)

Fungsi Actions(s) mengembalikan aksi yang tersedia pada keadaan s. Contoh: Actions(Arads) = {Go(Bucharest), Go(Sibiu), Go(Timisoara)}.

T Transition Model

Fungsi Result(s, a) mengembalikan keadaan setelah aksi a dilakukan pada s. Jika tidak diketahui, gunakan Transition(s, a) yang mengembalikan himpunan keadaan.

G Goal Test

Menentukan apakah keadaan saat ini adalah tujuan. Bisa berupa s == Goal (titik tunggal) atau s ∈ GoalSet (himpunan).

C Path Cost

Fungsi StepCost(s, a, s′) memberikan biaya translisi. Untuk state-space graph: bobot sisi. Untuk tree: biaya langkah.

Penting: Bentuk pencarian ditentukan oleh 6-tuple ini, bukan oleh 6-tuple asli dari masalah. Perbedaan ini memungkinkan agen menyelesaikan masalah yang berbeda dari masalah asli.

Rumusan Pencarian

State-space search = pencarian melalui graf keadaan. Pendekatan yang paling umum untuk sebagian besar masalah AI. Gunakan tree search atau graph search dengan 6-tuple masalah.

Breadth-First Search (Pencarian Melebar)

Memperluas node terdalam (paling dangkal) terlebih dahulu — mengunjungi semua node pada kedalaman d sebelum naik ke d+1.

\[ \text{BFS: Expand shallowest node in frontier} \]

Implementasi

Menggunakan FIFO queue: node yang diperluas lebih dulu dimasukkan lebih dulu ke frontier.

Analisis Kompleksitas

MetriksKompleksitas
Waktu (branching factor b, kedalaman m)\(O(b^{m+1})\) - kasus terburuk
Ruang\(O(b^{m+1})\): semua node di generasi saat ini
Kedalaman\(m + 1\)
Lengkap?Ya (jika b terbatas)
Optimal?Ya (jika biaya langkah sama/non-negatif)
Peringatan: BFS mengembalikan solusi terdangkal, bukan yang termurah jika biaya langkah berbeda.

Depth-First Search (Pencarian Mendalam)

Memperluas node paling dalam di frontier terlebih dahulu — selalu mengambil jalan paling kiri.

Implementasi

Menggunakan LIFO stack: expand node yang baru saja ditambahkan.

Analisis Kompleksitas

MetriksKompleksitas
Waktu (ruang keadaan terbatas b)\(O(b^m)\): node paling kiri paling dalam
Ruang\(O(bm)\): cukup satu jalur dari akar ke daun
Kedalaman\(m\)
Lengkap?Tidak (bisa masuk loop tak terbatas)
Optimal?Tidak (bisa menemukan solusi bukan termurah)
Mengapa DFS masih berguna? DFS menggunakan sangat sedikit memori (O(bm)), sehingga berguna untuk ruang keadaan yang sangat besar (miliaran state), asalkan solusi ditemukan di kedalaman awal.

Uniform-Cost Search (Pencarian Biaya Seragam)

Expand node dengan biaya jalur terendah g(n) — bukan yang paling dangkal.

\[ \text{UCS: Expand node with lowest } g(n) \]

Implementasi

Menggunakan priority queue berdasarkan g(n), urutan menaik. Mirip DFS, tapi selalu mengambil node termurah.

Analisis Kompleksitas

MetriksKompleksitas
Waktu\(O(b^{\lceil C^*/\varepsilon \rceil + 1})\): di mana \(C^*\) = biaya optimal, \(\varepsilon\) = biaya langkah minimum
RuangSama seperti waktu (priority queue)
Lengkap?Ya (jika \(\varepsilon > 0\))
Optimal?Ya — selalu menemukan jalur termurah
Contoh: DFS dapat memperluas node \(S \to A \to B \to C\) (biaya 50), sedangkan UCS memperluas \(S \to B\) (biaya 40) terlebih dahulu karena lebih murah.

Iterative Deepening Search (IDS)

Kombinasi DFS & BFS: menjalankan DFS hingga kedalaman limit \(l = 0, 1, 2, \ldots\) secara bertahap.

\[ \text{IDS = DFS(0), DFS(1), DFS(2), \ldots} \]

Analisis Kompleksitas

MetriksKompleksitas
Waktu\(O(b^m)\): DFS dominasi untuk depth \(m\)
Ruang\(O(bm)\): seperti DFS
Lengkap?Ya (jika \(b\) terbatas)
Optimal?Ya (jika biaya langkah sama)

Mengapa IDS Optimal?

  • DFS yang berlapis menjamin ditemukannya node di kedalaman \(d\) jika ada
  • Urutan penelusuran pada depth \(d\) identik dengan BFS
  • Sangat memori-efisien: \(O(bd)\)
Pertanyaan: Bukankah IDS redundan karena menjalankan DFS berulang kali? Jawaban: jumlah tambahan kerja relatif kecil, yaitu \((b + 1)/(b - 1)\) kali lebih banyak daripada satu kali DFS.

Latihan Terselesaikan

Latihan 1 — BFS vs DFS

Pada graf tak berarah berikut, tentukan urutan kunjungan BFS dan DFS dari node A ke H (asumsikan urutan leksikografis untuk tetangga):

A — B — C,   A — D — E,   C — F — H,   D — G — H

Tampilkan solusi

BFS (FIFO): A → B → D → C → E → F → G → H — menemukan H di kedalaman 3.

DFS (LIFO): A → B → C → F → H — menemukan H di kedalaman 4 (jalur A-B-C-F-H). Jika diimplementasikan dengan urutan tetangga leksikografis, DFS dapat menemukan jalur yang lebih panjang atau lebih pendek tergantung urutan.

Latihan 2 — Kompleksitas IDS

Mengapa IDS lebih disukai dari BFS untuk masalah dengan ruang keadaan yang sangat besar?

Tampilkan solusi

IDS memiliki kompleksitas memori O(bm) dibandingkan BFS yang memerlukan O(bd). Untuk masalah dengan b = 10 dan d = 20, BFS memerlukan ~1020 node di memori, sedangkan IDS hanya ~200 node. IDS sangat menghemat memori sambil tetap optimal (asalkan biaya langkah sama).

Latihan 3 — UCS dan Biaya Tidak Seragam

Jelaskan mengapa UCS optimal bahkan dengan biaya langkah yang berbeda-beda, sementara BFS tidak optimal.

Tampilkan solusi

UCS selalu expand node dengan g(n) terendah — artinya ia mengeksplorasi node berdasarkan biaya kumulatif, bukan kedalaman. BFS menganggap semua langkah sama biayanya, sehingga ketika biaya berbeda, BFS bisa menemukan jalur dangkal yang lebih mahal sebelum menemukan jalur lebih panjang yang lebih murah.
