Reference-point-based NSGA-III

NSGA-III diperkenalkan oleh Deb & Jain (2014) sebagai pengembangan dari NSGA-II yang dirancang khusus untuk masalah banyak objektif (many-objective) — lazimnya dipahami sebagai lebih dari tiga fungsi tujuan (NSGA-III diuji pada 3–15 objektif). Alih-alih mengandalkan crowding distance, NSGA-III menggunakan reference points yang tersebar secara sistematis untuk menjaga keragaman solusi pada Pareto front berdimensi tinggi.

Reference Points

Himpunan titik referensi yang tersebar merata pada hyperplane normalisasi. Setiap solusi dikaitkan ke titik referensi terdekat untuk menjaga penyebaran solusi di seluruh Pareto front.

📊

Non-dominated Sorting

Sama seperti NSGA-II, populasi diurutkan ke dalam front Pareto. Namun seleksi lingkungan menggunakan niching berdasarkan reference point, bukan crowding distance.

🔄

Normalisasi Adaptif

Setiap generasi, ruang objektif dinormalisasi menggunakan ideal point dan extreme points, sehingga reference points tetap relevan meskipun skala objektif berubah.

Langkah-langkah Algoritma NSGA-III

Kerangka utama NSGA-III mirip NSGA-II, tapi mekanisme seleksi lingkungan diganti dengan reference-point-based niching.

  1. Inisialisasi Bangkitkan populasi awal \(P_0\) berukuran \(N\). Bangkitkan himpunan reference points \(Z^r\) menggunakan metode Das-Dennis (simplex lattice design). Tetapkan \(t = 0\).
  2. Variasi (Crossover & Mutation) Terapkan SBX crossover dan polynomial mutation pada \(P_t\) untuk menghasilkan offspring \(Q_t\) berukuran \(N\).
  3. Non-dominated Sorting Gabungkan \(R_t = P_t \cup Q_t\) (ukuran \(2N\)). Urutkan \(R_t\) ke dalam front \(F_1, F_2, \dots\) berdasarkan dominasi Pareto.
  4. Normalisasi & Association Normalisasi nilai objektif setiap individu ke rentang [0,1] menggunakan ideal point dan extreme points. Lalu kaitkan setiap individu ke reference point terdekat dalam ruang normal (berdasarkan jarak tegak lurus / perpendicular distance).
  5. Niching & Seleksi Lingkungan Isi \(P_{t+1}\) dari front terbaik. Untuk front \(F_l\) yang tidak dapat ditampung seluruhnya, pilih individu berdasarkan niche count \(\rho_j\): prioritaskan reference point dengan \(\rho_j\) paling kecil. Jika seri, pilih individu dengan jarak tegak lurus terkecil ke reference point.
  6. Terminasi Jika \(t < T_{\max}\), \(t = t+1\) dan ulangi dari langkah 2. Jika tidak, kembalikan \(P_{t+1}\) sebagai hasil akhir.

Pseudo-code Ringkas

Input:  N (pop size), \(Z^r\) (reference points), T (max gen)
Output: P (final population)

1.  \(P_0\) ← init_population(N)
2.  for t = 0 to T-1:
3.    \(Q_t\) ← make_offspring(\(P_t\))  // SBX + polynomial mutation
4.    \(R_t\) ← \(P_t \cup Q_t\)
5.    F ← non_dominated_sort(\(R_t\))
6.    \(P_{t+1}\) ← ∅, i ← 1
7.    while |\(P_{t+1}\)| + |F_i| ≤ N:
8.      \(P_{t+1}\) ← \(P_{t+1} \cup F_i\); i++
9.    normalize(\(F_i\), \(Z^r\))
10.   associate(\(F_i\), \(Z^r\))  // hitung \(\rho_j\) & distance
11.   \(P_{t+1}\) ← niching_select(\(F_i\), \(Z^r\), \(\rho_j\), N)
12. return \(P_T\)
