Ringkasan lengkap Bab 4 buku Wheeled Mobile Robotics yang mencakup representasi lingkungan, perencanaan jalur berbasis sampling (RRT, PRM), algoritma Bug, dan metode pencarian berbasis graf (BFS, DFS, Dijkstra, A*). Setiap topik dilengkapi simulasi interaktif dan viewer PDF.

1

Representasi Lingkungan PF

Ruang konfigurasi robot harus dimodelkan sebelum pencarian jalur. Tiga pendekatan utama:

  • Roadmap — graf dari ruang bebas. Visibility graph menghubungkan simpul halangan yang saling terlihat → jalur terpendek namun menempel halangan. Voronoi diagram memaksimalkan jarak dari halangan → jalur lebih aman.
  • Cell Decomposition — ruang bebas dipecah menjadi sel. Exact (trapezoidal) mengikuti batas geometri; Approximate menggunakan grid seragam atau quadtree adaptif yang membagi sel hanya di daerah padat halangan.
  • Potential Field — medan potensial \(U(q)=U_{\text{attr}}+U_{\text{rep}}\). Robot mengikuti gradien negatif. Cepat namun rawan local minimum pada halangan cekung.

2

Potential Field Method PF

Robot bergerak dalam medan potensial buatan. Dua komponen gaya:

  • Gaya tarik (attractive) → menarik robot ke goal. \[U_{\text{attr}}(q)=\tfrac12 k_{\text{attr}}\,d^2(q,q_{\text{goal}})\quad\Rightarrow\quad F_{\text{attr}}=k_{\text{attr}}(q_{\text{goal}}-q)\]
  • Gaya tolak (repulsive) → menjauhkan robot dari halangan. \[U_{\text{rep}}(q)=\begin{cases}\tfrac12 k_{\text{rep}}\bigl(\frac1D-\frac1{D_0}\bigr)^2&D\le D_0\\0&D>D_0\end{cases}\] \[F_{\text{rep}}=k_{\text{rep}}\bigl(\frac1D-\frac1{D_0}\bigr)\frac1{D^2}\,(q-q_{\text{obst}})\]
  • Robot mengikuti resultan \(F_{\text{total}} = F_{\text{attr}} + F_{\text{rep}}\). Masalah: local minimum pada halangan cekung → robot berhenti sebelum mencapai goal.

3

Bug Algorithms Bug

Algoritma paling sederhana: hanya butuh sensor jarak lokal dan posisi goal global. Dua perilaku dasar: gerak lurus ke goal dan boundary following (ikuti kontur halangan).

  • Bug0 — lurus ke goal. Jika terhalang, ikuti kontur (kiri) sampai bisa lurus lagi. Paling sederhana, tapi tidak complete — bisa gagal pada halangan tertentu.
  • Bug1 — saat terhalang, telusuri seluruh kontur, catat titik terdekat ke goal, lalu kembali ke sana. Complete — selalu menemukan path jika ada. Panjang path \(\le \frac32 L_{\text{obst}} + d_{\text{Euclid}}\).
  • Bug2 — ikuti main line (garis start–goal). Saat terhalang, ikuti kontur sampai main line tercapai lagi dengan jarak lebih dekat. Lebih efisien dari Bug1, tapi bisa memutar halangan berulang kali.

4

RRT RRT

Rapidly-exploring Random Tree — metode sampling-based untuk pencarian path dari satu start ke satu goal:

  • Tree dimulai dari \(q_{\text{start}}\). Pada setiap iterasi: (a) sampel titik acak \(q_{\text{rand}}\), (b) cari node terdekat \(q_{\text{near}}\) di tree, (c) ekstensi sejauh \(\varepsilon\) ke arah \(q_{\text{rand}}\) → \(q_{\text{new}}\), (d) jika \(q_{\text{new}}\) dan koneksinya di ruang bebas, tambahkan ke tree.
  • Goal bias: dengan probabilitas tertentu (mis. 10%), \(q_{\text{rand}}\) diganti dengan \(q_{\text{goal}}\) untuk mempercepat konvergensi.
  • Hanya dua parameter: ukuran langkah \(\varepsilon\) dan jumlah iterasi. Tree cepat meluas ke area yang belum dijelajahi (Gbr. 4.18 di paper).
  • Efektif untuk ruang konfigurasi dimensi tinggi; probabilistically complete.

5

PRM PRM

Probabilistic Roadmap — metode sampling-based untuk banyak query (multi-start, multi-goal):

  • Learning phase: bangun graf (roadmap) dengan mengambil \(N\) sampel acak di ruang bebas, lalu hubungkan setiap sampel ke \(K\) tetangga terdekat dalam radius \(D\). Semua edge diperiksa tabrakannya.
  • Query phase: hubungkan \(q_{\text{start}}\) dan \(q_{\text{goal}}\) ke node terdekat di graf, lalu cari path (misal BFS/A*).
  • Narrow passage: masalah umum PRM — daerah sempit jarang tersampel. Solusi: bridge test — tiga titik acak pada garis lurus; jika titik luar di halangan dan titik tengah di ruang bebas, tambahkan sebagai node.

6

Uninformed Graph Search GS

Pencarian tanpa informasi tambahan selain definisi masalah. Menggunakan open list (node hidup) dan closed list (node sudah diperiksa):

  • BFS (Breadth-First Search) — open list sebagai FIFO. Eksplorasi shallow nodes dulu. Complete — menemukan path dengan jumlah langkah terkecil. Tidak optimal jika biaya transisi bervariasi. Memori besar.
  • DFS (Depth-First Search) — open list sebagai LIFO. Eksplorasi node terdalam dulu. Tidak complete (bisa terjebak siklus/cabang tak terbatas). Memori kecil. Tidak optimal.
  • Iterative Deepening DFS — DFS dengan batas kedalaman dinaikkan bertahap. Complete, memori kecil, optimal jika biaya seragam. Kombinasi kelebihan BFS + DFS.

7

Informed Graph Search GS

Pencarian dengan fungsi heuristik \(h(n)\) yang memperkirakan biaya dari node \(n\) ke goal:

  • Dijkstra — open list diurutkan berdasarkan \(g(n)\) (cost-to-here). Complete & optimal untuk bobot positif. Temukan jarak terpendek ke semua node. Bisa dihentikan lebih awal jika goal sudah di-closed.
  • A* — open list diurutkan berdasarkan \(f(n)=g(n)+h(n)\). Complete & optimal jika \(h\) admissible (optimistic, tidak melebihbiayai). Heuristik umum: Euclidean distance, Manhattan distance. Lebih efisien dari Dijkstra karena terfokus ke goal.
  • Greedy Best-First Search — open list diurutkan hanya berdasarkan \(h(n)\). Sangat cepat (langsung ke goal), tapi tidak optimal — bisa menghasilkan path lebih panjang. Heuristik tidak perlu admissible.
