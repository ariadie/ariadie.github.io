Formulasi Masalah Optimasi (Problem Definition)

Dalam Pymoo, mendefinisikan masalah optimasi kustom dilakukan dengan cara membuat subclass baru dari modul pymoo.core.problem. Pymoo menyediakan beberapa kelas basis tergantung pada bagaimana Anda ingin mengevaluasi solusi.

Dua Metode Utama Evaluasi

👤 Elementwise Evaluation

Evaluasi solusi satu per satu solusi. Menggunakan kelas basis ElementwiseProblem. Sangat mudah ditulis dan didebug, tetapi memiliki overhead loop Python yang tinggi jika populasi besar.

⚡ Vectorized Evaluation

Evaluasi populasi secara serentak dalam bentuk matriks 2D menggunakan NumPy. Menggunakan kelas basis Problem. Jauh lebih cepat karena memanfaatkan komputasi ter-vektor, tetapi membutuhkan penulisan kode berbasis matriks.

Parameter Inisialisasi Masalah

Saat memanggil konstruktor kelas basis super().__init__(), parameter berikut wajib diisi:

  • n_var: Jumlah variabel keputusan.
  • n_obj: Jumlah fungsi objektif (tujuan) yang ingin diminimalkan.
  • n_ieq_constr: Jumlah kendala pertidaksamaan ($g(x) \le 0$).
  • n_eq_constr: Jumlah kendala persamaan ($h(x) = 0$).
  • xl: Batas bawah variabel (Lower Bounds) dalam bentuk NumPy array berukuran n_var.
  • xu: Batas atas variabel (Upper Bounds) dalam bentuk NumPy array berukuran n_var.

1. Implementasi Menggunakan ElementwiseProblem

Pendekatan ini mengevaluasi satu larik 1D x. Nilai objektif disimpan ke out["F"] dan kendala ke out["G"].

from pymoo.core.problem import ElementwiseProblem
import numpy as np

class MyElementwiseProblem(ElementwiseProblem):
    def __init__(self):
        super().__init__(n_var=2, n_obj=1, n_ieq_constr=1, 
                         xl=np.array([-5.0, -5.0]), 
                         xu=np.array([5.0, 5.0]))

    def _evaluate(self, x, out, *args, **kwargs):
        # x berupa array 1D berisi [x1, x2]
        out["F"] = x[0]**2 + x[1]**2
        out["G"] = x[0] + x[1] - 1.0 # Kendala: x1 + x2 <= 1.0

2. Implementasi Menggunakan Vectorized Problem

Pendekatan ini mengevaluasi seluruh populasi sekaligus. Input X berupa matriks 2D berukuran (N, n_var) di mana N adalah ukuran populasi.

from pymoo.core.problem import Problem
import numpy as np

class MyVectorizedProblem(Problem):
    def __init__(self):
        super().__init__(n_var=2, n_obj=1, n_ieq_constr=1, 
                         xl=np.array([-5.0, -5.0]), 
                         xu=np.array([5.0, 5.0]))

    def _evaluate(self, X, out, *args, **kwargs):
        # X adalah matriks 2D shape (pop_size, 2)
        # Mengevaluasi semua individu secara efisien menggunakan NumPy
        out["F"] = X[:, 0]**2 + X[:, 1]**2
        out["G"] = X[:, 0] + X[:, 1] - 1.0
💡

Rekomendasi Kecepatan: Gunakan ElementwiseProblem saat awal pengembangan agar Anda dapat fokus merancang logika kendala. Pindahlah ke Problem yang ter-vektor jika Anda membutuhkan optimasi berskala besar dengan populasi ribuan atau evaluasi generasi yang lama.

Tipe Variabel Keputusan Lainnya

Pymoo mendukung struktur variabel non-kontinu secara bawaan, seperti:

  • Variabel Biner: Diimplementasikan dengan membatasi ruang pencarian menggunakan representasi operator biner khusus.
  • Variabel Permutasi: Cocok untuk Traveling Salesman Problem (TSP), dioptimalkan menggunakan operator permutasi khusus.
  • Variabel Campuran (Mixed-Variable): Pymoo mendukung pencampuran variabel integer, real, biner, dan kategori melalui konfigurasi operator kustom.

Anda dapat mengonfigurasi tipe variabel keputusan dengan mengimpor operator sampling dan mutasi yang sesuai (misal: BinaryRandomSampling untuk biner).
