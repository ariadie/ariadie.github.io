Daftar Algoritme Optimasi di Pymoo

Pymoo menyediakan banyak algoritme optimasi yang dirancang untuk berbagai kategori masalah (Single, Multi, dan Many-objective) dan tipe variabel (Kontinu, Diskret, Biner, Permutasi, dll.). Berikut adalah ringkasan algoritme yang paling umum digunakan.

Tabel Ringkasan Algoritme Utama

Algoritme Nama Kelas Kategori Kendala? Deskripsi Singkat
Genetic Algorithm (GA) GA Single-objective Ya Algoritme genetika modular klasik yang sangat mudah dikustomisasi untuk berbagai tipe representasi variabel.
Differential Evolution (DE) DE Single-objective Ya Variasi mutasi berbasis diferensial vektor, sangat efektif untuk optimasi numerik kontinu skala global.
Particle Swarm (PSO) PSO Single-objective Ya Kecerdasan kelompok di mana partikel bergerak dipandu oleh posisi terbaik pribadi dan kelompok.
CMA-ES CMAES Single-objective Tidak Algoritme berbasis model kovariansi matriks untuk pencarian lokal/global kontinu tanpa turunan.
NSGA-II NSGA2 Multi-objective Ya Standar industri MOO dengan pemilahan non-dominasi (non-dominated sorting) dan crowding distance untuk keberagaman solusi.
SPEA2 SPEA2 Multi-objective Ya Menggunakan k-th nearest neighbor density estimation dan mekanisme pemotongan arsip (archive truncation) untuk keberagaman.
NSGA-III NSGA3 Many-objective Ya Penyempurnaan NSGA-II untuk tujuan ≥ 3, menggunakan titik acuan (reference points) daripada crowding distance.
MOEA/D MOEAD Many-objective Tidak Mendekomposisi masalah multiobjektif menjadi sejumlah submasalah single-objective yang dioptimalkan secara paralel.
AGE-MOEA / AGE-MOEA2 AGEMOEA Many-objective Tidak NSGA-II yang dimodifikasi dengan estimasi geometri Pareto front untuk memandu seleksi alam secara dinamis.

Contoh Konfigurasi Algoritme Kustom

1. Mengonfigurasi Differential Evolution (DE)

DE sangat cocok untuk optimasi bernilai kontinu. Anda bisa menentukan strategi mutasi (seperti DE_rand_1_bin) dan nilai crossover:

from pymoo.algorithms.soo.de import DE
from pymoo.operators.crossover.dex import DEX

algorithm = DE(
    pop_size=50,
    variant="DE/rand/1/bin",
    CR=0.9,
    F=0.8,
    crossover=DEX(bias=0.5)
)

2. Mengonfigurasi NSGA-III (Many-objective Optimization)

Untuk optimasi many-objective (3 atau lebih tujuan), crowding distance pada NSGA-II kehilangan efektifitasnya. NSGA-III menggunakan titik-titik arah acuan (reference directions) untuk mempartisi ruang objektif:

from pymoo.algorithms.moo.nsga3 import NSGA3
from pymoo.util.ref_dirs import get_reference_directions

# Membuat reference directions di atas unit simplex (misal: 3 objektif)
ref_dirs = get_reference_directions("das-dennis", 3, n_partitions=12)

algorithm = NSGA3(
    pop_size=92, # Ukuran populasi disesuaikan dengan jumlah arah acuan
    ref_dirs=ref_dirs
)
💡

Cara Memilih Algoritme: Jika tujuan Anda hanya 1, gunakan GA atau DE. Jika tujuan Anda 2, gunakan NSGA2. Jika tujuan Anda 3 atau lebih, gunakan NSGA3 atau MOEAD.
